About Project

Project name: Better-Ordering-Food
Purpose:
  1. ทราบคิวเมนูอาหารที่มีอยู่ในขณะนั้น
  2. ไม่ต้องมารับออเดอร์จากลูกค้าด้วยตนเอง
PROS:
  1. ลดความผิดพลาดในการสั่งเมนูอาหาร
  2. ลดปัญหาการข้ามคิว
  3. ใช้งานง่าย สะดวก
CONS:
  1. ไม่สามารถดูข้อมูลผ่านทางออนไลน์ได้
  2. ต้องใช้ไฟเลี้ยงตลอดเวลา

Source Code: Here!
Detail: เนื่องจากการสั่งอาหารในบางครั้งมีความผิดพลาดในการสื่อสารระหว่างผู้รับบริการและผู้ให้บริการ ทำให้ผู้รับบริการอาจไม่ได้รับอาหารตามที่ต้องการ ซึ่งอาจส่งผลเสียทั้งสองฝ่ายในเรื่องของอารมณ์และทรัพยากรโครงงาน Better-Ordering-Food จึงจัดทำขึ้นโดยมีวัตถุประสงค์เพื่อพัฒนาการสั่งอาหารให้มีประสิทธิภาพมากยิ่งขึ้น โดยเป็นการลดความผิดพลาดที่อาจเกิดขึ้นในการสื่อสารระหว่างผู้รับบริการและผู้ให้บริการ อีกทั้งยังช่วยลดปัญหาการถกเถียง โต้แย้ง และความไม่พึงพอใจจากการสื่อสารที่ไม่ถูกต้อง ผู้พัฒนาจึงได้จัดทำโครงงานนี้ขึ้นเพื่อประยุกต์ใช้ความรู้ที่ได้จากการเรียนเรื่อง C Programming และ Micro-controller ซึ่งเป็นส่วนหนึ่งของวิชา Computer Programming

อุปกรณ์ที่ใช้:
  1. Arduino Uno R3 แบบ SMD + สาย Usb
  2. หน้าจอแสดงผล LCD1602 IIC/I2C Green Backlight
  3. 4x4 Matrix Keypad
  4. Passive Buzzer Module
  5. สายจัมป์ 20 ซม. ผู้-ผู้
  6. สายจัมป์ 20 ซม. ผู้-เมีย
